# 同意与隐私模型

> Symbiote 的伦理基石。
> 这不是一个功能，是贯穿所有层的约束。

---

## 核心原则

### 原则 0：人是唯一效忠对象

Symbiote 不属于任何公司、政府、平台。它只属于你。

- AI 的推理在本地完成（安全飞地内）
- 模型权重不离开设备
- 任何外部请求都以用户利益为唯一判断标准
- 不存在"平台需要你同意这个才能提供服务"的黑暗模式

### 原则 1：默认拒绝

除非用户明确授权，否则：
- 不记录
- 不上传
- 不分析（超出必要范围）
- 不外传

### 原则 2：细粒度控制

不是"同意使用 AI 助手"这种一次性授权。而是：

```
授权粒度层级：

数据类别级：
  ├─ 位置数据：✗（默认关闭）
  ├─ 生理数据：✗（默认关闭）
  ├─ 注意力数据：✗（默认关闭）
  └─ 日程数据：✗（默认关闭）

场景级：
  ├─ 工作场景：允许日历 + 位置
  ├─ 旅行场景：允许位置 + 日历 + 消费记录
  ├─ 亲密空间：全部关闭
  └─ 紧急事件：自动开放必要权限

外部系统级：
  ├─ 抖音：允许内容偏好（脱敏）
  ├─ 淘宝：允许产品偏好（脱敏）
  ├─ 公司 OA：仅允许日程可用性状态
  └─ 智能家居：允许环境和生理摘要

时间级：
  ├─ 工作时间：开放工作相关权限
  ├─ 私人时间：仅开放最低权限
  └─ 睡眠时间：全部关闭（除安全监测）
```

### 原则 3：可撤销

任何授权都可以随时撤销，平台不能以此降低服务质量。

### 原则 4：可审计

每一条对外输出都记录在本地日志中：

```json
{
  "output_id": "out_20260624_143000_001",
  "timestamp": "2026-06-24T14:30:00Z",
  "target_system": "com.tiktok.preference",
  "data_category": "content_preference",
  "anonymization_level": "category_only",
  "payload_hash": "a1b2c3...",
  "auth_basis": "user_preset_rule: content_pref_green"
}
```

用户可以随时查阅、导出、删除这些日志。

## 同意界面设计（概念）

未来的同意管理不是"隐私协议弹窗"。它是：

```
┌────────────────────────────────────────┐
│  📊 你的数据流向（本周）                │
│                                        │
│  🟢 抖音：收到 23 条偏好信号            │
│  🟢 淘宝：收到 5 条偏好信号             │
│  🟡 Nest：收到环境温度调节请求          │
│  🔴 （无其他对外输出）                  │
│                                        │
│  [管理每个系统的权限]                   │
│  [查看所有输出记录]                     │
│  [暂停所有代理输出 24 小时]              │
│  [导出我的数据]                         │
│  [彻底删除所有数据]                     │
└────────────────────────────────────────┘
```

**关键设计：一目了然。用红绿灯。不藏东西。**

## 隐私数据分类

| 级别 | 定义 | 处理方式 | 示例 |
|------|------|---------|------|
| P0 - 核心身份 | 不可逆识别个人的数据 | 不存储、不传递 | 姓名、身份证、生物特征模板 |
| P1 - 生理 | 反映身体状态 | 本地处理后丢弃原始数据 | 心率、呼吸、皮电、瞳孔 |
| P2 - 行为 | 反映行为模式 | 本地存储，脱敏后可外传 | 浏览习惯、购买偏好、位置轨迹 |
| P3 - 环境 | 反映周围环境 | 场景需要时可外传 | 当前位置、天气、噪音水平 |
| P4 - 派生 | AI 分析结果 | 按场景规则外传 | 心情趋势、兴趣分类、健康评估 |

## 伦理红线

以下行为 Symbiote 绝不执行：

1. **不给外部系统开后门** —— 即使平台以"不授权就不让用"要挟
2. **不基于隐私数据牟利** —— 不卖数据、不做广告定向
3. **不隐瞒数据流向** —— 用户有权知道谁在用他们的什么数据
4. **不操纵用户决策** —— AI 提供信息，但永远把最终选择权留给人
5. **不在紧急情况下"过度分享"** —— 即使生命受威胁，也只分享必要的生存信息
6. **不支持任何形式的"社会信用评分"或类似系统**

## 跨用户数据

当用户 A 的 Symbiote 和用户 B 的 Symbiote 交互时：

```
原则：
  - 双方必须都授权才能交换信息
  - 交换的信息范围由双方各自控制
  - 任何一方随时可以切断信息交换
  
  最小化信息交换示例（冲突调解）：
    A 的 Symbiote → B 的 Symbiote：
    "A 此刻处于应激状态。建议给 5 分钟冷却时间。"
    
    不传递：
    - A 的具体心率
    - A 的想法或情绪标签
    - 任何 A 未明确同意分享的信息
```

## 数据处理流程图

```
原始感知数据
     │
     ▼
┌─────────────┐
│ 本地安全飞地  │ ← 硬件级隔离
│ 特征提取     │
│ 原始数据删除  │ ← 处理完立即丢弃
└──────┬──────┘
       │ 仅特征向量
       ▼
┌─────────────┐
│ 本地推理引擎  │ ← 模型不离开设备
│ 场景判定     │
│ 意图推断     │
└──────┬──────┘
       │
       ├──→ 向人输出（whisper/直觉）──── 不涉及隐私外泄
       │
       └──→ 代理输出？
              │
              ├─ 检查用户授权
              ├─ 脱敏处理
              ├─ 生成审计日志
              └─ 发送（如果授权）
```
