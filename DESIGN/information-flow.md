# 信息流设计

> Symbiote 中数据的生命周期：从环境到 AI，从 AI 到人和外界。

---

## 信息流全景

```
                    ┌─────────────────────────┐
                    │      环境                │
                    │  GPS 音频 日历 IoT 生理  │
                    └───────────┬─────────────┘
                                │ 多模态信号 (100+ Hz)
                                ▼
              ┌─────────────────────────────────┐
              │         感知层                   │
              │  信号→特征提取→感知帧             │
              │  原始数据在此层之后即被丢弃        │
              └─────────────┬───────────────────┘
                            │ 感知帧 (10 Hz)
                            ▼
              ┌─────────────────────────────────┐
              │       意图引擎                   │
              │  场景分类→意图推断→约束判定→仲裁   │
              └─────────────┬───────────────────┘
                            │ 意图帧 (事件驱动)
                            ▼
              ┌─────────────────────────────────┐
              │      人格适配层                  │
              │  模式选择→语气控制→输出决策       │
              └──────┬──────────────────┬───────┘
                     │                  │
        向人输出     │                  │  向外界输出
                     ▼                  ▼
        ┌──────────────────┐  ┌──────────────────┐
        │  神经接口层       │  │  代理输出层       │
        │  whisper/直觉/    │  │  偏好声明/状态广播│
        │  感受/flash       │  │  意图传达/隐私请求│
        └────────┬─────────┘  └────────┬─────────┘
                 │                     │
                 ▼                     ▼
            ┌─────────┐         ┌──────────────┐
            │  人类    │         │  外部系统      │
            └─────────┘         └──────────────┘
```

## 数据时效性

| 数据类型 | 生命周期 | 存储位置 | 删除策略 |
|---------|---------|---------|---------|
| 原始感知信号 | < 100ms | 内存缓冲区 | 处理后立即清除 |
| 感知帧 | < 5 分钟 | 本地安全飞地 | 意图帧生成后丢弃 |
| 意图帧 | < 24 小时 | 本地加密存储 | 每日滚动清除 |
| 长期行为模式 | 用户决定 | 本地 + 可选云端 | 用户随时可删除 |
| 代理输出日志 | 永久（用户可控） | 本地加密存储 | 用户随时可导出并删除 |

## 信息不对称原则

Symbiote 故意维护一种信息不对称——偏向用户的：

```
AI 知道的 > 外部平台知道的

AI 拥有：
  ├─ 完整的感知数据（本地）
  ├─ 准确的意图推断
  └─ 用户的历史偏好

外部平台收到：
  ├─ 脱敏的偏好类别
  ├─ 用户明确授权的内容
  └─ 无法反推出原始数据

→ "数据优势"属于用户，不属于平台。
```

---

## 反馈回路

用户对 AI 输出的反应会形成反馈，持续优化后续推断：

```
AI 输出 → 用户接受/拒绝/修正 → 记录偏好 → 下次更准确
```

- 如果用户连续 3 次拒绝某类建议 → AI 降低该类输出的频率
- 如果用户在某种场景下总是修正 AI 的语气 → AI 记住该场景的语气偏好
- 反馈学习完全在本地完成，不依赖云端
