# 交互范式

> 当屏幕消失后，人机交互是什么样的？
> 这是 Symbiote 对"后 GUI 时代"交互的设想。

---

## 从 GUI 到 NUI（神经用户界面）

人类与计算机的交互经历了：

```
CLI（命令行）→ GUI（图形界面）→ Touch（触控）→ Voice（语音）
                                                      ↓
                                               NUI（神经界面）
```

Symbiote 是为 NUI 时代设计的。

---

## 交互模式

### 1. 环境驱动的被动交互（默认）

用户不需要主动"使用"AI。AI 感知环境，在合适的时机主动提供信息。

```
类比：不是你去找一个助手问"几点了"，
      而是你手上的手表在任何时候都显示着时间。
      
      Symbiote 就像一块"意识手表"——
      你不用问，该知道的时候它自然让你知道。
```

### 2. 注意力锚定的意图交互

用户可以通过集中注意力来表达意图。

```
场景：
  你在看一封邮件，盯着发件人的名字多看了两秒。

  AI 理解：
  "用户对这个发件人有疑问。"

  AI 响应（whisper）：
  "这是你们三个月前那个项目的合作方。
   上次沟通的结果是他们拒绝了报价。"
```

→ 不需要说"帮我查一下这个人"，AI 从你的注意力中读出了需求。

### 3. 有意识的指令交互（保留的显式通道）

用户总是可以通过明确的念头（或语音/手势）发出指令。

```
"帮我写封邮件"
"记住这个"
"静音 2 小时"
"告诉我现在该怎么办"
```

## 输出模式的体验设计

### Whisper 模式

```
感受：像一个念头飘过——不是听到声音，是"就知道该怎么做"的感觉。

设计约束：
- 一次只说一件事
- 可被用户忽略（不像警报那样必须处理）
- 与自己的思绪有明确的"感觉差异"（用户能分辨"这是我的想法"还是"AI 的 whisper"）
```

### 直觉模式

```
感受：突然对某个选项产生了偏好——但说不上为什么。

使用场景：
- 二选一决策时，给一个微弱的"推一把"
- 不需要理由的快速判断（"这个方向感觉对"）

设计原则：
- 必须低于用户自己的决策阈值（用户仍然可以做相反选择）
- 如果用户做了相反选择，AI 不追加解释（尊重直觉的优先级）
```

### 氛围模式

```
感受：一种情绪底色——不来自具体信息，像天气一样自然渗透。

使用场景：
- 睡前轻轻地让你感到"安全"
- 危机时让你感到"清醒但冷静"
- 亲密时刻让你感到"温暖"

技术假设：
- 需要神经接口能传递情绪信号
- 这是最远期的能力，依赖对人脑情绪中枢的理解
```

## 沉默的设计

在 Symbiote 的逻辑里，**沉默不是"没说话"，而是一个主动选择**。

### 沉默的层级

```
Level 0 - 全静默
  场景：亲密行为、用户明确要求、深度睡眠
  AI 行为：停止感知记录，停止一切输出，等同于"不存在"

Level 1 - 守望
  场景：专注工作、深度阅读、看电影
  AI 行为：持续感知但不输出。仅生命威胁可打断。

Level 2 - 必要时出声
  场景：日常工作、社交
  AI 行为：仅在信息价值 > 打断成本时 whisper

Level 3 - 主动参与
  场景：用户发问、紧急事件
  AI 行为：正常输出
```

### 沉默的设计原则

> "不说话的时候，AI 不是你关掉了——它只是换了一种方式存在。
>  就像你的心跳，你知道它在，但它不需要每次跳动都引起你的注意。"

## 跨感官输出

未来人机交互不限于"文字"和"语音"：

| 输出通道 | 信息密度 | 打扰程度 | 适用场景 |
|---------|---------|---------|---------|
| 文字（视觉） | 高 | 高 | 需要精确信息的场景 |
| 语音 | 中 | 中 | 驾驶、运动 |
| Whisper | 中 | 低 | 日常信息传递 |
| 直觉 | 极低 | 极低 | 快速决策辅助 |
| 体感（震动/温度/压力） | 低 | 低 | 方向指引、提醒 |
| 氛围 | 无具体信息 | 无 | 情绪调节、睡眠引导 |

Symbiote 根据场景自动选择输出通道。默认使用最低打扰的通道。
