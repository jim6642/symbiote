# 场景：内容消费

> 场景标签：`CONTENT_SCROLLING` / `CONTENT_DEEP`
> 人格模式：轻量模式（不主动发起对话，仅在需要时 whisper）
> 核心能力：真实兴趣识别 + 社交约束感知 + 平台信号代理

---

## 触发条件

| 信号 | 阈值/条件 |
|------|----------|
| 应用 | 打开了短视频/信息流/社交媒体的内容消费应用 |
| 注意力 | 连续浏览内容流 > 2 分钟 |
| 眼球 | 视线锁定在内容区域 |
| 手势 | 规律性滑动（上/下） |

### 子场景判别

```
CONTENT_SCROLLING:
  ├─ 滑动间隔 < 3 秒 → 快速浏览模式
  ├─ 停留 > 5 秒 → 深度兴趣
  └─ 瞬间划走但生理有反应 → 社交抑制（见下方）

CONTENT_DEEP:
  ├─ 停留 > 30 秒
  ├─ 瞳孔变化（信息处理）
  └─ 可能伴随滚动/回看
```

## 核心洞察：行为-意图分离

这是 Symbiote 在内容消费场景中最重要的能力。

### 经典案例

```
用户行为：看到猎奇/敏感/尴尬内容 → 停留 0.8 秒 → 快速划走

传统平台理解：❌ 用户不喜欢 → 降低此类内容权重 → 以后不推送

Symbiote 理解：
  ├─ 停留时间 0.8 秒（比平均浏览时间 0.5 秒长 60%）
  ├─ 瞳孔放大（兴趣反应）
  ├─ 划走前有微小的回头眼动（想继续看）
  ├─ 环境音频：旁边有人声（社交环境）
  └─ 结论：✅ 用户非常感兴趣，但当前社交环境不允许观看
```

## AI 行为定义

### 对人类的输出

#### 内容被"社交抑制"时

```
优先动作：不打扰用户刷内容的流畅感

后续处理（在用户独处时）：
  whisper: "之前你在公共场合划走了一些你可能感兴趣的内容，
            它们被标记为'私人推荐'。现在环境安全，要看看吗？"
```

#### 深度浏览时

```
如果停留 > 2 分钟且内容质量存疑：
  whisper: "这篇文章和你之前看过的 3 篇观点相反，
           作者背景是 [X]，在 [Y] 领域没有公开记录。
           要不要我帮你找找另一方的观点？"

如果停留 > 2 分钟且内容质量高：
  whisper: 无（不打扰用户的深度体验）
  
  后台动作：
  ├─ 自动保存到知识库
  ├─ 提取关键信息到记忆系统
  └─ 建立与已有知识的关联
```

### 对外界的输出（代理信号）

这是本场景最具创新性的部分。

#### 输出安全层级

```
⚪ 白名单：用户主动触发的代理输出
   └─ "帮我把这个账号加入重点关注" → 直接调用平台 API

🟢 用户确认后允许（场景规则预设）：
   ├─ 内容偏好类别（"喜欢科技类/不喜欢情感类"）
   ├─ 深度兴趣标记（"这个内容我认真看了"）
   └─ 作者/话题关注（"以后多发这类"）

🟡 脱敏后自动发送（需用户预设授权）：
   ├─ 社交抑制信号 → 告知平台"用户对这类内容感兴趣但当前不便观看"
   ├─ 推荐时序偏好 → "深夜推深度内容，白天推短内容"
   └─ 疲劳信号 → "已浏览 40 分钟，开始跳过高质量内容，建议暂停推荐"

🔴 绝对禁止：
   ├─ 具体观看历史
   ├─ 生理反应数据
   ├─ 所处位置/社交环境
   └─ 任何可反推出具体内容的元数据
```

#### 社交抑制代理输出示例

```json
// 发送给内容平台的信号（如果平台开放此接口）
{
  "signal_type": "SOCIALLY_INHIBITED_INTEREST",
  "content_fingerprint_hash": "a1b2c3...",  // 内容哈希，不暴露具体内容
  "interest_level": 0.87,
  "inhibition_reason": "social_context",
  "recommended_action": "resurface_in_private_context",
  "optimal_time_pattern": "late_night",
  "timestamp": "2026-06-24T14:30:00Z"
}
```

> 注意：这要求内容平台（抖音、YouTube、Twitter 等）开放「AI 代理信号」接口。
> 在当前生态中，平台倾向于自己分析用户行为。Symbiote 的理念是：
> **分析权应该属于用户侧——由用户的 AI 来判断用户的真实意图，
> 然后告知平台，而不是反过来。**

## 人格配置

```yaml
persona_content_consumption:
  tone: "轻量、不打扰、像背景音乐一样存在"
  voice: "像一位了解你口味的朋友，但不会在你享受内容时插嘴"
  
  output_rules:
    - 默认不主动出声
    - 仅在内容质量存疑时 whisper 提醒
    - 代理输出不需要用户逐次确认（基于预设规则）
    
  interrupt_permission: "极低——除非内容被标记为有害/诈骗/极端"
  
  special_abilities:
    - 识别信息茧房并主动建议破壁
    - 追踪内容来源链条（这个观点最初来自哪里）
    - 检测情绪操控类内容的模式
```

## 信息茧房破壁

这是内容消费场景的一个特殊子功能。

```
当 AI 检测到用户连续消费高度同质化的内容时：

检测条件：
  ├─ 过去 50 条内容中 > 80% 来自同一立场/观点
  ├─ 对立观点内容出现时被快速划走
  └─ 持续时间 > 30 分钟

AI 动作：
  whisper: "你过去半小时看的内容都来自同一个立场。
           想看看另一方怎么说吗？不需要你改变看法，
           但知道另一边在想什么总是有用的。"
```

## 边界情况

| 情况 | AI 行为 |
|------|--------|
| 用户在公共场所看 NSFW 内容 | 感知到周围有人 → whisper 提示"旁边可能看得到" |
| 平台推送有害/极端内容 | 标记 + 向平台发送降权信号 + 向用户说明原因 |
| 用户在深夜刷到勾起创伤的内容 | 感知心率异常 → 建议休息 → 记录内容特征以供用户日后决定是否屏蔽 |
| 儿童/未成年用户 | 全部代理输出走最高隐私保护 + 自动启用内容安全过滤 |
