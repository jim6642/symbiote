# Symbiote 系统架构

> 版本: v0.1.0-concept
> 状态: 概念蓝图

---

## 架构总览

Symbiote 由五个逻辑层组成，数据从环境流入，经过层层处理后，以两种形式流出：向人输出（whisper/直觉）和向外界输出（代理意图）。

```
┌──────────────────────────────────────────────────────┐
│                    ① 感知层                           │
│  GPS │ 日历 │ 消息 │ 音频 │ 生理 │ 眼球 │ IoT │ 社交图  │
└────────────────────┬─────────────────────────────────┘
                     │ 原始信号流 (100+ Hz)
                     ▼
┌──────────────────────────────────────────────────────┐
│                  ② 意图引擎                           │
│  场景分类器 → 意图推断器 → 社交约束模型 → 优先级仲裁   │
└────────────────────┬─────────────────────────────────┘
                     │ 场景标签 + 意图向量 + 优先级
                     ▼
┌──────────────────────────────────────────────────────┐
│                  ③ 人格适配层                         │
│  语气选择 → 输出格式决策 → 时机判断 → 详略控制        │
└──────┬───────────────────────────────┬───────────────┘
       │ 向人输出                       │ 向外界输出
       ▼                               ▼
┌──────────────┐             ┌──────────────────────┐
│ ④ 神经接口层 │             │ ⑤ 代理输出层          │
│ whisper      │             │ 平台 API 调用         │
│ 直觉信号     │             │ 隐私过滤             │
│ 感受传递     │             │ 审计日志             │
└──────┬───────┘             └──────┬───────────────┘
       │                            │
       ▼                            ▼
   ┌────────┐                ┌──────────────┐
   │ 人类   │                │ 外部系统      │
   └────────┘                │(抖音/淘宝/... )│
                             └──────────────┘
```

## ① 感知层 (Perception Layer)

### 职责
持续采集多模态环境数据和生理数据，输出统一格式的「感知帧」(Perception Frame)。

### 数据源分类

| 类别 | 数据源 | 采样频率 | 隐私敏感度 |
|------|--------|---------|-----------|
| 🛰 时空 | GPS、WiFi 指纹、气压计 | 1-10 Hz | 🟡 中 |
| 📅 日程 | 日历、邮件、消息 | 事件驱动 | 🟡 中 |
| 👁 注意力 | 眼球追踪、瞳孔直径、眨眼频率 | 60-120 Hz | 🔴 高 |
| 💓 生理 | 心率、HRV、皮电、呼吸、体温 | 1-100 Hz | 🔴 高 |
| 🎤 环境音频 | 对话内容、语气、环境噪声 | 16-48 kHz | 🔴 高 |
| 🏠 IoT | 智能家居、车辆传感器 | 事件驱动 | 🟡 中 |
| 🌐 社交图 | 通讯录、社交网络关系 | 低频更新 | 🟡 中 |

### 感知帧结构

```protobuf
message PerceptionFrame {
  uint64 timestamp_ms = 1;
  GeoLocation location = 2;
  CalendarState calendar = 3;
  AttentionMetrics attention = 4;
  PhysiologicalMetrics physiology = 5;
  AudioContext audio = 6;
  IoTContext iot = 7;
  SocialContext social = 8;
}
```

### 设计约束
- 所有数据在本地完成第一轮处理后即丢弃原始信号
- 生理和音频数据**默认不离开设备**
- 向上层传递的是**特征向量**而非原始数据

## ② 意图引擎 (Intent Engine)

### 职责
从感知帧中推断「当前场景是什么」「用户真实意图是什么」「社交约束是什么」，输出统一的「意图帧」。

### 核心组件

#### 2.1 场景分类器 (Scene Classifier)

将感知帧映射到一个或多个场景标签：

```
场景标签体系（非互斥，可叠加）：

🏢 WORK_MEETING      工作会议
🏢 WORK_SOLO          独自工作
🏢 WORK_PRESENTATION  正在汇报/演讲
✈️ TRAVEL_EXPLORING   自由探索
✈️ TRAVEL_TRANSIT     交通工具上
💑 INTIMATE_PRIVATE   私密空间
💑 INTIMATE_CONFLICT  关系紧张
👥 SOCIAL_FORMAL      正式社交
👥 SOCIAL_CASUAL      休闲社交
📱 CONTENT_SCROLLING  刷内容流
📱 CONTENT_DEEP       深度浏览
🛒 SHOPPING_BROWSING  随便逛
🛒 SHOPPING_INTENT    有目标的购物
📚 LEARNING_FOCUSED   专注学习
📚 LEARNING_FATIGUED  学习疲劳
🚨 CRISIS_PHYSICAL    身体危险
🚨 CRISIS_MEDICAL     医疗紧急
🌙 SLEEP_LIGHT        浅睡
🌙 SLEEP_DEEP         深睡
😶 IDLE               空闲/放空
```

#### 2.2 意图推断器 (Intent Inferrer)

从行为中推断真实意图，核心能力是识别**行为-意图分离**：

```
行为                    →  真实意图（由 AI 推断）
─────────────────────────────────────────────────
快速划走猎奇视频         →  感兴趣但社交环境不安全
盯着商品看了 3 秒后关掉  →  想要但在犹豫/嫌贵
会议中沉默点头           →  有异议但不方便说
对伴侣说"没事"           →  有事但不想现在谈
深夜反复翻身             →  焦虑/失眠，非物理不适
```

实现方法（概念）：多模态 transformer，输入为感知帧序列 + 历史行为嵌入，输出为意图向量 + 置信度。

#### 2.3 社交约束模型 (Social Constraint Model)

判断当前环境允许什么样的 AI 行为：

| 约束维度 | 示例 |
|---------|------|
| 👀 视觉隐私 | 旁边有人能看到屏幕 → 不显示敏感内容 |
| 🔊 音频隐私 | 周围安静 → 不发出声音，使用 whisper |
| 👤 社交形象 | 在老板面前 → 不暴露真实态度 |
| ⏰ 时间适宜 | 凌晨 3 点 → 非紧急不打扰 |
| 📍 地点敏感 | 在会议室 → 代理输出切换到静默模式 |

#### 2.4 优先级仲裁器 (Priority Arbiter)

当多个意图/场景同时触发时，决定优先级：

```
优先级排序规则：
1. 🚨 人身安全 > 一切
2. 💓 严重的情绪危机 > 工作/社交
3. ⏰ 有时限的决策 > 无时限
4. 👤 当前对话对象的优先级 > 背景事件
5. 🤖 用户主动发起的 > AI 推测的
```

### 意图帧结构

```protobuf
message IntentFrame {
  uint64 timestamp_ms = 1;
  repeated SceneLabel scenes = 2;
  IntentVector user_intent = 3;
  SocialConstraint constraints = 4;
  Priority priority = 5;
  float confidence = 6;  // 0.0 - 1.0
}
```

## ③ 人格适配层 (Persona Adapter)

### 职责
根据场景标签和社交约束，选择输出的人格模式、语气、信息密度的详细程度。

### 人格模式定义

每种模式是一个输出风格配置：

```yaml
persona_work:
  tone: "干练、理性、数据驱动"
  greeting: 无（直接进入主题）
  sentence_length: 中短
  emotional_range: 克制
  output_preference:
    - 结构化数据优先
    - 概率和置信度必附
    - 允许打断当前任务
  example: "B 方案成功概率 73%。需要提前和财务部对齐预算。要我现在起草邮件吗？"

persona_travel:
  tone: "热情、有故事感、引导式"
  greeting: 环境感知（"到了！"）
  sentence_length: 中长
  emotional_range: 丰富
  output_preference:
    - 叙事 > 数据
    - 提供选择而非指令
    - 保持探索感
  example: "你右手边那条巷子深处有家 30 年的老店——他们家的招牌在这个城市排前三。想拐进去看看吗？"

persona_intimate:
  tone: "温柔、克制、不越界"
  greeting: 感知情绪后决定是否出声
  sentence_length: 短
  emotional_range: 深度但克制
  output_preference:
    - 沉默是默认动作
    - 回应 > 主动
    - 不使用量化语言
  example: "我在。"（然后沉默）

persona_crisis:
  tone: "冷静、果断、短句"
  greeting: 无
  sentence_length: 极短
  emotional_range: 零——完全理性
  output_preference:
    - 指令式
    - 一次只说一件事
    - 每条输出带上确定感
  example: "现在离开这个路口。向右走。快。"
```

### 模式切换逻辑

```
输入: IntentFrame
  ↓
匹配场景标签 → 选择基础 persona
  ↓
叠加社交约束 → 调整输出方式（音量/媒介/详略）
  ↓
检查用户历史偏好 → 微调（比如用户讨厌被打断）
  ↓
输出: PersonaConfig（给到下游的输出层）
```

## ④ 神经接口层 (Neural Interface Layer)

### 职责
将 AI 输出转化为人脑可接收的形式。**这是整个架构中技术最遥远、依赖脑机接口成熟度的部分。**

### 输出模式

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **Whisper** | 内在语言——像在脑子里听到一个声音，但不是听觉 | 日常信息传递 |
| **Intuition** | 微弱的"感觉"——对某事产生莫名的偏好或不安 | 决策辅助，不需要理由时 |
| **Feeling** | 情绪渲染——AI 注入一种氛围感 | 亲密空间、危机警觉 |
| **Flash** | 瞬间的知识浮现——像突然想起一件事 | 记忆辅助、快速参考 |
| **Silent** | 什么都不传递——AI 保持静默 | 用户专注/休息/亲密时刻 |

### 接口假设

本层依赖尚未存在的技术：
- 高带宽、双向的神经接口（读写脑皮层信号）
- 精确到单个神经元集群的刺激传递
- 安全的神经编码/解码协议

因此本节为**远期占位**，不做详细技术规格。

## ⑤ 代理输出层 (Proxy Output Layer)

### 职责
以用户的身份，向外部系统发送经过隐私过滤和用户授权的意图信号。

### 输出原则

1. **用户授权优先**：没有授权，绝不外传任何信息。
2. **脱敏优先**：能传偏好就不传行为，能传类别就不传内容。
3. **可撤销**：用户随时可以撤销某个系统或某类数据的授权。
4. **可审计**：每一条对外输出都记录在本地日志中，用户可查阅。

### 输出示例

```
场景: 用户刷到猎奇视频，喜欢但快速划走

AI 推断: 感兴趣 + 社交环境不安全
↓
检查授权: 用户已授权「内容偏好」类别可向抖音匿名传递
↓
构建输出:
{
  "signal": "INTERESTED_BUT_SOCIALLY_BLOCKED",
  "content_category": "curiosity_edge",
  "recommend_later": true,
  "recommend_context": "private_browsing",
  "user_id": null,  // 匿名
  "timestamp": "2026-06-24T22:15:00Z"
}
↓
发送至: 抖音偏好 API（如果抖音未来开放此类接口）
```

### 输出安全层级

```
⚪ 白名单：总是允许
   └─ 用户主动说"告诉它 X" 的直接指令

🟢 用户确认后允许：
   ├─ 产品偏好（哪类商品、什么价位）
   ├─ 内容偏好（喜欢什么类型、什么风格）
   └─ 需求信号（正在找什么）

🟡 脱敏后允许：
   ├─ 聚合统计数据（"80% 的用户喜欢 X" 而非 "张三喜欢 X"）
   └─ 经过差分隐私处理的行为模式

🔴 绝对禁止：
   ├─ 生理数据
   ├─ 情感状态
   ├─ 位置历史
   ├─ 对话内容
   └─ 任何可追溯到具体个人的行为日志
```

## 数据流时序

一次完整的「感知 → 推断 → 输出」周期：

```
T+0ms    感知帧到达
T+5ms    场景分类完成
T+8ms    意图推断完成
T+10ms   社交约束判定完成
T+12ms   优先级仲裁完成
T+15ms   人格模式匹配完成
T+18ms   输出决策完成
T+20ms   向人类输出（whisper/直觉）─┐
T+22ms   向外界输出（如果授权）     ├─ 并行
T+25ms   审计日志写入完成          │
         ← 全程 < 30ms ──────────┘
```

> 这是远期目标。当前技术条件下，仅 LLM 推理就需数百毫秒到数秒。
> 此处的数字代表「当硬件和算法成熟时应达到的响应水平」——足够快才能融入意识流。

## 安全架构

### 数据隔离

```
┌──────────────────────────────┐
│  本地安全飞地 (Secure Enclave) │
│  ┌──────────────────────────┐│
│  │ 原始感知数据（短期缓冲）    ││
│  │ → 处理后立即删除          ││
│  └──────────────────────────┘│
│  ┌──────────────────────────┐│
│  │ 特征向量（长期存储）        ││
│  │ → 加密静态存储            ││
│  └──────────────────────────┘│
│  ┌──────────────────────────┐│
│  │ 模型权重（本地推理）        ││
│  │ → 永不离开设备            ││
│  └──────────────────────────┘│
└──────────────────────────────┘
         │
         │ 仅特征向量（脱敏后）
         ▼
┌──────────────────────────────┐
│  云端（可选，用户控制）         │
│  ┌──────────────────────────┐│
│  │ 长周期模式学习             ││
│  │ 跨设备同步                ││
│  │ 联邦学习贡献（如果用户允许） ││
│  └──────────────────────────┘│
└──────────────────────────────┘
```

### 攻击面

| 攻击 | 风险 | 缓解措施 |
|------|------|---------|
| 原始感知数据泄露 | 🔴 致命 | 本地处理后即丢弃，不上传 |
| 代理输出被劫持 | 🔴 致命 | 输出签名 + 授权票据 |
| 人格模式被恶意切换 | 🟡 严重 | 模式切换需生物特征确认 |
| 外部平台滥用代理数据 | 🟡 严重 | 每次输出附带用途限制声明 |
| 脑机接口物理攻击 | 🔴 致命 | 硬件级安全飞地 |

## 演进路线

```
Phase 1（当下）: 决策预设原型
  └─ 通过文字接口模拟「场景感知 + prompt 包装」
  └─ 见 PROTOTYPE/decision-preset/

Phase 2（近期）: 多模态感知雏形
  └─ 集成现有传感器（手机 GPS + 日历 + 消息）
  └─ 基于规则引擎的场景判定

Phase 3（中期）: 意图引擎 + 人格适配
  └─ LLM 驱动的场景理解和意图推断
  └─ 人格模式切换的端到端实现

Phase 4（远期）: 神经接口
  └─ 依赖脑机接口技术成熟
  └─ whisper/直觉输出模式

Phase 5（远期）: 代理输出生态
  └─ 外部平台开放「AI 代理」API
  └─ 标准化意图输出协议
```
