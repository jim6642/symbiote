# 贡献指南

Symbiote 是一个概念蓝图项目。贡献不限于代码。

---

## 贡献方式

### 📝 讨论 & 反馈
- 在 Issues 中提出你的想法、质疑、批判
- 对现有文档的改进建议
- 你认为遗漏的场景

### 🎨 场景设计
- 按照 `SCENARIOS/` 中的模板，设计新的 AI 行为场景
- 提交到 `SCENARIOS/` 目录

### 💻 原型开发
- 改进 `PROTOTYPE/decision-preset/` 的代码
- 添加新的 prompt 模板
- 实现其他今天可行的局部原型

### 📐 规格完善
- 完善 `SPECS/` 中的技术规格
- 添加更详细的数据结构定义
- 补充缺失的接口

### 🌍 翻译
- 将文档翻译为更多语言
- 确保中文和英文版本同步

---

## 设计哲学（请先读这个）

Symbiote 不是一个"功能越多越好"的项目。提交前请检查：

1. **是否提高了用户的自主权？**
   — 好的功能让用户更有掌控感，坏的功能代用户做了决定。

2. **是否保护了隐私？**
   — 默认不收集、不上传、不外传。

3. **AI 的存在感是否恰当？**
   — 不是所有时候都需要 AI 说话。沉默是一种功能。

4. **是否能用今天的任何技术做到？**
   — 纯蓝图部分放在 SPECS/ 和 DESIGN/，能实现的放在 PROTOTYPE/。

## Pull Request 流程

1. Fork 项目
2. 创建你的功能分支 (`git checkout -b feature/amazing-idea`)
3. 提交你的改动 (`git commit -m 'Add amazing idea'`)
4. 推送到分支 (`git push origin feature/amazing-idea`)
5. 打开 Pull Request

---

## 行为准则

- 尊重所有贡献者的时间和想法
- 建设性讨论，不人身攻击
- 这是一个关于未来的项目——保持好奇心
